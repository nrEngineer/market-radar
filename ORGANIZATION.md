## 2. 各部署詳細

### Dept.1 フロントエンド開発部 (Frontend Development)

**ミッション**: 金融・コンサルティング業界水準のプロフェッショナルUIを実現し、データの価値を最大化するユーザー体験を提供する

**責任範囲**:
| 領域 | 具体的業務 |
|------|-----------|
| UI/UX設計・実装 | Figmaデザイン → React/Next.js実装、デザインシステム管理 |
| フォーマルUI実現 | **ダークモード完全廃止**、ライトテーマ・プロフェッショナルデザイン |
| レスポンシブ対応 | モバイル/タブレット/デスクトップの最適化、PWA対応 |
| パフォーマンス最適化 | Core Web Vitals（LCP < 2.5s, FID < 100ms, CLS < 0.1） |
| アクセシビリティ | WCAG 2.1 AA準拠、スクリーンリーダー対応 |
| コンポーネント設計 | 再利用可能なUIコンポーネントライブラリの構築・保守 |

**フォーマルUI設計要件**:
- ✅ ダークモード完全廃止
- ✅ ライトテーマ（白背景ベース）・プロフェッショナル
- ✅ 金融・コンサルティング業界標準デザイン（McKinsey/BCG/Bloomberg風）
- ✅ 洗練されたタイポグラフィ（Inter + Noto Sans JP）
- ✅ ビジネスカラーパレット（ネイビー/スレートブルー基調）
- ✅ フォーマルなレイアウト・十分な余白設計
- ✅ データ可視化の高品質化（Chart.js → D3.js移行検討）

**KPI**:
| 指標 | 目標値 | 測定方法 |
|------|--------|---------|
| Lighthouse Performance | ≥ 90 | CI/CD自動測定 |
| Lighthouse Accessibility | ≥ 95 | CI/CD自動測定 |
| デザインシステム採用率 | 100% | コンポーネント使用率 |
| UI不具合件数 | ≤ 3件/月 | バグトラッカー |
| ユーザビリティテストスコア | ≥ 85/100 | 月次テスト |

### Dept.2 バックエンド開発部 (Backend Development)

**ミッション**: 高信頼性・高パフォーマンスのAPI基盤を構築し、フロントエンドとデータ層を最適に接続する

**責任範囲**:
| 領域 | 具体的業務 |
|------|-----------|
| API設計・実装 | RESTful/GraphQL API設計、バージョニング、ドキュメント自動生成 |
| データベース設計 | Supabase PostgreSQLスキーマ設計、マイグレーション管理、インデックス最適化 |
| サーバーサイドロジック | Next.js API Routes、Edge Functions、Cron Jobs |
| パフォーマンス最適化 | クエリ最適化、キャッシュ戦略（Redis/Vercel Edge Cache）、N+1問題解消 |
| 認証・認可 | JWT/OAuth2.0、RBAC、Supabase Auth統合 |

**KPI**:
| 指標 | 目標値 | 測定方法 |
|------|--------|---------|
| API応答時間 (P95) | ≤ 200ms | APMツール |
| API稼働率 | ≥ 99.9% | Uptime Monitor |
| エンドポイントカバレッジ | 100% | API仕様書 vs 実装 |
| DBクエリ実行時間 (P95) | ≤ 50ms | スロークエリログ |
| テストカバレッジ | ≥ 80% | Jest/Vitest |

### Dept.4 AI・分析エンジン部 (AI & Analytics Engine)

**ミッション**: LLMと統計モデルを統合した分析エンジンにより、コンサルティングファーム品質の洞察を自動生成する

**責任範囲**:
| 領域 | 具体的業務 |
|------|-----------|
| LLM統合・最適化 | OpenAI/Claude API統合、プロンプトエンジニアリング、コスト最適化 |
| スコアリングモデル | 機会スコアリング（市場規模×成長性×競合×実現性×タイミング×モート） |
| 予測アルゴリズム | トレンド予測（ARIMA/Prophet）、市場規模予測、需要予測 |
| 機械学習パイプライン | モデル学習・評価・デプロイ、A/Bテスト |
| 自動TAM/SAM/SOM推計 | 公開データからの段階的市場規模推計、クロスバリデーション |

**KPI**:
| 指標 | 目標値 | 測定方法 |
|------|--------|---------|
| 洞察具体性スコア | ≥ 8/10 | 専門家レビュー |
| 予測精度 (MAPE) | ≤ 20% | バックテスト |
| スコアリング再現可能性 | ≥ 95% | 同一入力→同一出力率 |
| LLMコスト/分析 | ≤ $0.05 | API使用量 |
| 分析生成速度 | ≤ 30秒 | レスポンスタイム |

### Dept.6 セキュリティ部 (Security) ★新設

**ミッション**: プロダクト全体のセキュリティを統括し、脆弱性ゼロ・データ保護100%を実現する

**責任範囲**:
| 領域 | 具体的業務 |
|------|-----------|
| セキュリティ監査 | コードレビュー、依存パッケージ監査、OWASP Top 10チェック |
| 脆弱性スキャン | SAST/DAST自動スキャン、ペネトレーションテスト（四半期） |
| アクセス制御 | API認証強化、RBAC設計、シークレット管理（Vault） |
| データ保護 | 暗号化（転送中/保存時）、PII管理、データ分類 |
| コンプライアンス | 個人情報保護法対応、利用規約・プライバシーポリシー管理 |
| インシデント対応 | セキュリティインシデント対応プラン、フォレンジック手順 |

**KPI**:
| 指標 | 目標値 | 測定方法 |
|------|--------|---------|
| 重大脆弱性数 | 0件 | セキュリティスキャン |
| シークレット漏洩数 | 0件 | GitGuardian等 |
| 依存パッケージ脆弱性 | 0件(Critical) | npm audit / Snyk |
| セキュリティ監査頻度 | 月次 | 監査ログ |
| インシデント対応時間 | ≤ 1時間 | インシデントログ |

**即時対応タスク**:
- [ ] ハードコードトークン (`'Bearer ***REMOVED***'`) の完全除去
- [ ] 全API Routeの認証実装
- [ ] 環境変数の適切な分離（.env.local → Vercel環境変数）
- [ ] CSP (Content Security Policy) ヘッダー設定
- [ ] Rate Limiting実装
