# 🔄 Git管理プロトコル - CEO Claw規則

## 📋 開発・Git管理必須ルール

### ⚡ 毎回実行必須
```bash
# 1. 実装後即座実行
git add .
git status

# 2. 意味のあるコミットメッセージ
git commit -m "🎯 [FEATURE] 具体的な機能追加・修正内容
- 変更点1
- 変更点2  
- 影響・効果"

# 3. 即座GitHub同期
git push origin main
```

### 📊 コミット頻度
- **機能実装後**: 即座コミット・プッシュ
- **PDCA実行後**: 結果・改善をコミット
- **バグ修正後**: 修正内容をコミット  
- **日次終了時**: 必ず全変更プッシュ確認

### 🎯 コミットメッセージ規則
```
🚀 [NEW] 新機能追加
🔧 [FIX] バグ修正  
📈 [IMPROVE] 機能改善・最適化
🌐 [SEO] SEO・国際化対応
💾 [DATA] データベース・PDCA関連
📋 [DOC] ドキュメント・レビュー
⚡ [HOTFIX] 緊急修正
```

### 🔍 毎日チェック項目
- [ ] 未コミットファイル確認: `git status`
- [ ] 最新プッシュ確認: `git log --oneline -5`
- [ ] GitHub同期確認: リモート状況確認
- [ ] ブランチ状況確認: `git branch -v`
